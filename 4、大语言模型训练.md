## 4.1 LLM的定义理解
LLM，即 Large Language Model，中文名为大语言模型或大型语言模型，是一种相较传统语言模型参数量更多、在更大规模语料上进行预训练的语言模型。一般来说，LLM 指包含**数百亿（或更多）参数的语言模型**，它们往往在**数 T token 语料上**通过多卡分布式集群进行预训练，具备远超出传统预训练模型的文本理解与生成能力。只要模型展现出**涌现能力**，即在一系列复杂任务上表现出远超传统预训练模型（如 BERT、T5）的能力与潜力，都可以称之为 LLM。

### LLM的能力
1. 涌现能力：
	区分 LLM 与传统 PLM 最显著的特征即是 LLM 具备 **涌现能力**。涌现能力是指同样的模型架构与预训练任务下，某些能力在小型模型中不明显，但在大型模型中特别突出。涌现能力的显现就像是模型性能随着规模增大而迅速提升，超过了随机水平。
2. 上下文学习：
	上下文学习能力是由 GPT-3 首次引入的。具体而言，上下文学习是指允许语言模型在提供自然语言指令或多个任务示例的情况下，通过理解上下文并生成相应输出的方式来执行任务，而无需额外的训练或参数更新。
3. 指令遵循：
	通过使用自然语言描述的多任务数据进行微调，也就是所谓的**指令微调**。经过指令微调的 LLM 能够理解并遵循未见过的指令，并根据任务指令执行任务，而无需事先见过具体示例，展现出大语言模型的泛化能力
4. 逐步推理：
	LLM 通过采用思维链（Chain-of-Thought，CoT）推理策略，可以利用包含中间推理步骤的提示机制来解决这些任务，从而得出最终答案。据推测，这种能力可能是通过对代码的训练获得的。

### LLM的特点
1. 多语言支持
2. 长文本处理
3. 拓展多模态
4. 幻觉问题

## 4.2 训练LLM
一般而言，训练一个完整的LLM需要经过三个阶段：——PreTrain、SFT和RLHF。
### PreTrain
预训练，是训练LLM最核心也是工程量最大的一步。LLM的预训练和传统预训练模型非常类似，同样是使用海量无监督文本对随机初始化的模型参数进行训练。目前主流的LLM几乎都i采用的Decoder-Only的类GPT架构（Llama架构），它们的预训练任务特都是沿乘了GPT模型的经典预训练任务——因果语言模型（Casual Language Model，CLM：即和最初的语言模型一致，通过给出上下文要求模型预测下一个token来进行训练）。LLM的预训练同传统预训练模型的核心差异在于，预训练的体量和资源消耗。
	- 为了满足加速训练模型的目的，分布式训练框架也成为了LLM训练必不可少的组成部分。分布式训练框架的核心思路就是数据并行和模型并行；数据并行，是指训练模型的尺寸可以被当GPU内存容纳，但是由于增大训练的Batch_size会增大显存开销，无法使用较大的batch_size进行训练，同时下，训练数据飞非常大，使用单张GPU训练时长也是会很漫长。
### SFT
监督微调。LLM所覆盖的海量知识基本都是源于预训练语料。LLM 的性能本身，核心也在于预训练的工作。SFT 的主要目标是让模型从多种类型、多种风格的指令中获得泛化的指令遵循能力，也就是能够理解并回复用户的指令。因此，类似于 Pretrain，SFT 的数据质量和数据配比也是决定模型指令遵循能力的重要因素。
	一般SFT所使用的指令数据集包括以下三个键：
	```python
	{
		'instruction':"输入的用户指令“，
		'input':"执行该指令可能需要的补充输入，没有设置为空"，
		'output':"模型给出的回复"
	}
	```
	同时，为使模型可以学到和预训练不同的范式，在SFT的过程中，往往会针对性设置特定格式。
	```python
	### Instruction:\n{{content}}\n\n### Response:\n
	```
其中content即为具体的用户指令，也就是说，对于每一个用户指令，将会嵌入到上文的content部分，这里的用户指令不仅仅是上例中的'instruction'，而是指令和输入的拼接，即模型可以执行的一条完整的指令。
！！！ 注意，指令微调本质上还是对模型进行CLM训练，只不过是要求模型对指令进行理解和回复而不是简单预测下一个token，所以模型预测的结果不仅是output，而应该是input+output,只不过input部分是不需要去参与loss计算的，但是回复指令本身还是以预测下一个token的形式实现的。但是，随着LLM能力的不断增强，模型的多轮对话能力逐渐受到重视。所谓的多轮对话，是指，=模型在每一次的对话时可以参考之前的历史记录来做出回复。
模型是否支持多轮对话，与预训练是没有关系的。事实上，模型的多轮对话完全来自于SFT阶段。如果要使模型支持多轮对话，就需要在SFT阶段将训练数据构造成多轮对话格式，让模型可以利用之前的知识来生成回答。
构造多轮样本一般有三种方式：
 1. 直接将最后一次模型回复作为输出，前面的所有历史对话作为输入，直接拟合最后一次回复：
	 ```python
	  input=<prompt_1><completion_1><prompt_2><completion_2><prompt_3><completion_3>
	 output=[MASK][MASK][MASK][MASK][MASK]<completion_3>
	 ```
2. 将 N 轮对话构造成 N 个样本：
 ```python
 input_1 = <prompt_1><completion_1>
 output_1 = [MASK]<completion_1>

 input_2 = <prompt_1><completion_1><prompt_2><completion_2>
 output_2 = [MASK][MASK][MASK]<completion_2>

 input_3=<prompt_1><completion_1><prompt_2><completion_2><prompt_3><completion_3>
 output_3=[MASK][MASK][MASK][MASK][MASK]<completion_3>
 ```
 3. 直接要求模型预测每一轮对话的输出：
 ```python
  input=<prompt_1><completion_1><prompt_2><completion_2><prompt_3><completion_3>
  output=[MASK]<completion_1>[MASK]<completion_2>[MASK]<completion_3>
 ```
### RLHF
Reinforcement Learning from Human Feedback，即人类反馈强化学习，是利用强化学习来训练 LLM 的关键步骤。RLHF 的思路是，引入强化学习的技术，通过实时的人类反馈令 LLM 能够给出更令人类满意的回复。强化学习是有别于监督学习的另一种机器学习方法，主要讨论的问题是智能体怎么在复杂、不确定的环境中最大化它能获得的奖励。强化学习主要由两部分构成：智能体和环境。在强化学习过程中，智能体会不断行动并从环境获取反馈，根据反馈来调整自己行动的策略。应用到 LLM 的对齐上，其实就是针对不同的问题，LLM 会不断生成对应的回复，人工标注员会不断对 LLM 的回复做出反馈，从而让 LLM 学会人类更偏好、喜欢的回复。
	 - RLHF分为两个步骤：训练RM和PPO训练。
		 奖励模型（Reward Model，RM）的主要作用是学习人类对不同模型回复的偏好，并在后续强化学习阶段为大语言模型生成的回复提供奖励信号。在 PPO 训练过程中，策略模型针对给定的用户指令生成回复，奖励模型则对该回复进行评估，并输出一个标量奖励值。该奖励值用于衡量模型回复与人类偏好的符合程度，策略模型根据奖励信号更新参数，从而逐步提高生成结果的有用性、真实性和安全性。
		 从模型结构上看，奖励模型通常以预训练语言模型为基础，并在模型最后一层隐藏状态之上增加一个标量奖励头。对于输入的“指令—回复”序列，奖励模型将其映射为一个实数分数。因此，奖励模型在结构上与文本分类模型较为相似，但二者的输出目标并不相同：传统文本分类模型通常输出不同类别的概率分布，而奖励模型直接输出一个连续的标量分数，用于表示回复的相对偏好程度。
		 在奖励模型的训练过程中，通常不直接要求人工标注者为每条回复给出绝对的数值奖励。一方面，不同标注者对评分尺度的理解可能存在差异；另一方面，即使标注者对回复优劣具有一致判断，也可能给出差异较大的具体分数。直接使用绝对分数进行监督训练，容易将标注者之间的尺度差异引入奖励模型，使模型难以学习稳定的人类偏好。
		 因此，RLHF 通常采用相对偏好标注方式。对于同一个用户指令，首先由语言模型生成多个候选回复，再由人工标注者对这些回复进行排序，或者从两个候选回复中选择更优的一个。根据排序结果，构造偏好数据对。
